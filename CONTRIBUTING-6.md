# Contributing to DEXHUNTER

Thanks for considering a contribution. This project is early and moving fast, so a few ground rules keep things clean.

## Getting Started

1. Fork the repo
2. Create a branch: `git checkout -b feature/short-description`
3. Make your changes
4. Commit with a clear message: `git commit -m "Add: short description"`
5. Push and open a Pull Request

## Branch Naming

- `feature/...` for new features
- `fix/...` for bug fixes
- `refactor/...` for non-functional changes

## Code Style

- TypeScript everywhere, avoid `any` unless unavoidable
- Functional components with hooks, no class components
- Tailwind utility classes for styling, no separate CSS files unless global
- Keep components readable. Split large components rather than growing one file indefinitely

## Pull Requests

- Describe what changed and why
- Reference any related issue
- Screenshots or screen recordings are appreciated for UI changes

## Questions

Open an issue or reach out directly.
